# Support

Use the repository's structured issue forms:

- **Bug report** for reproducible incorrect runtime or type behavior.
- **Feature or API proposal** for new operators and workflows.
- **Performance report** for measured speed, memory, allocation, startup, or
  bundle-size regressions.
- **Documentation improvement** for missing or unclear guidance.
- **Usage question** for a focused integration question with a concrete schema.

General debugging requests without a reproduction may be closed until the
required details are available. Never include credentials, private datasets, or
unredacted personal information. Security reports must follow
[SECURITY.md](SECURITY.md) and must not be filed publicly.
