This index mirrors the public namespace users import from
`@jit-compiler/jit/runtime`. Every runtime member must have a destination here.
The site test suite compares this table with `Object.keys(JIT)` and verifies
every linked MDX file, so a new export cannot silently ship without
documentation.

## Runtime or AOT?

All high-level compilers share one optimizer and emitter. Runtime mode pays
code generation once and caches the resulting function by schema identity.
AOT performs that work during `jit generate` and writes an import-free module.
Use `/runtime` for dynamic schemas and development convenience; use `/define`
plus generated imports when startup cost, CSP or client bundle weight matters.
